Wavy/Home

Redesigned Wavy's homepage to improve event navigation and accessibility. We added a List View with Swimlanes and a Timeline for easier sorting and event tracking, making it the default view. The Calendar View was updated for better spacing and readability with accessible, pastel themes. Differentiated event types through distinct visual styles and icons for quick identification. Lastly, renamed 'Culture Pal' to 'Smart Suggestions', making it clearer and more intuitive for users.

1.00 Problem

1.00 Problem

Wavy’s homepage was a single calendar view showing events, notable days, and suggestions, but it had issues. 1/ Calendar items weren’t labeled, so you couldn’t quickly spot invites or RSVPs. 2/ You could only see one month at a time, making it easy to forget events months ahead and hard to find past ones. 3/ Optional themes had either harsh, inaccessible colors or made the text almost unreadable.

2.00 List View

2.00 List View

We added a List View to the homepage, organizing events into Swimlanes and a Timeline. Swimlanes categorize events, like Yours/Others, Upcoming/Past, or Invites/RSVPs, while the Timeline shows all events in chronological order. We made this the default view because most users come to the dashboard looking for a specific event (“I need to edit my event” or “What do I need to bring for Cocktail Making?”), which was harder to find on the calendar.

3.00 Calendar View

3.00 Calendar View

We refreshed the Calendar View to maximize space by reducing text size, item padding, and cell spacing, fitting twice as many items. We also updated the themes to a WACG-compliant pastel palette, which reinforced Wavy’s playful brand and, you know, just looked better.

4.00 Differenciated Styles

4.00 Differenciated Styles

The old layout had little visual difference between events, notable days, and suggestions, making it hard to tell them apart. We needed each item to have its own visual language that worked across all surfaces, including calendar themes, swimlanes, and “Your events” sections. We gave events a white background with black text, notable days a colored background with colored text, and suggestions a transparent background with a dashed outline and gray text. Events were further distinguished by an icon system showing their status: Invited, Going, Declined, or Went. This made it easier to quickly identify relevant events, especially in the calendar and timeline.

5.00 Smart Suggestions

5.00 Smart Suggestions

The toggle for show notable days and suggestions was named “Culture Pal,” which confused users about its purpose. To fix this, we renamed Culture Pal to Smart Suggestions, making it clear that the toggle adds contextual information to the layout. We used a gray sparkle and dashed lines as visual cues for suggestions. This design worked across both List and Calendar views, allowing users to easily spot suggestions wherever they appeared.
